## חלק 1: העלאה ל-Apple App Store

### שלב 1: פתיחת חשבון Apple Developer

1. היכנסו ל-[developer.apple.com](https://developer.apple.com)
2. לחצו על "Account" ואז "Enroll"
3. בחרו סוג חשבון:
   - **Individual**: לעצמאיים ויחידים
   - **Organization**: לחברות (דורש מספר D-U-N-S)
4. שלמו $99
5. המתינו לאישור (1-3 ימי עסקים)

> 💡 **טיפ**: אם אתם חברה, הכינו את מספר ה-D-U-N-S מראש. הנפקה לוקחת עד שבועיים!

### שלב 2: הכנת האפליקציה ב-Xcode

1. ודאו שה-Bundle Identifier ייחודי
2. הגדירו את גרסת האפליקציה ומספר Build
3. בחרו את ה-Team (חשבון Developer שלכם)
4. הגדירו Signing & Capabilities
5. בנו את האפליקציה ל-Archive

```
Product → Archive → Distribute App → App Store Connect
```

### שלב 3: יצירת רשומה ב-App Store Connect

1. היכנסו ל-[appstoreconnect.apple.com](https://appstoreconnect.apple.com)
2. לחצו על "My Apps" → "+" → "New App"
3. מלאו את הפרטים:

| שדה | דוגמה |
|-----|-------|
| Platform | iOS |
| Name | שם האפליקציה |
| Primary Language | Hebrew |
| Bundle ID | com.yourcompany.appname |
| SKU | APPNAME-2025 |

### שלב 4: העלאת צילומי מסך

**גדלים נדרשים (2025):**

| מכשיר | גודל (פיקסלים) | חובה? |
|-------|---------------|--------|
| iPhone 6.9" | 1320 x 2868 | כן |
| iPhone 6.7" | 1290 x 2796 | כן |
| iPhone 6.5" | 1284 x 2778 | לא (אבל מומלץ) |
| iPhone 5.5" | 1242 x 2208 | לא |
| iPad Pro 13" | 2064 x 2752 | אם תומך באייפד |
| iPad Pro 12.9" | 2048 x 2732 | אם תומך באייפד |

**כללים לצילומי מסך:**
- 3-10 צילומים לכל גודל
- אסור לכלול מחירים שאינם ממכשיר אמיתי
- אסור להראות מסכי תשלום של צד שלישי
- מותר להוסיף טקסט שיווקי מעל הצילום

### שלב 5: מילוי מטא-דאטה

**תיאור קצר (Subtitle):**
- עד 30 תווים
- מופיע מתחת לשם באפליקציה
- דוגמה: "הזמנת אוכל בקליק"

**מילות מפתח (Keywords):**
- עד 100 תווים, מופרדות בפסיקים
- אל תחזרו על שם האפליקציה
- השתמשו במילים נפוצות בעברית

**תיאור מלא:**
- עד 4,000 תווים
- 3 השורות הראשונות הכי חשובות (מופיעות לפני "עוד")
- כללו: מה האפליקציה עושה, למי היא מיועדת, תכונות עיקריות

### שלב 6: הגדרות נוספות

**דירוג גיל:**
- ענו על השאלון בכנות
- תוכן למבוגרים = 17+
- תוכן לכל הגילאים = 4+

**מדיניות פרטיות:**
- חובה לכל אפליקציה
- צריך להיות URL נגיש
- צריך לפרט מה אוספים ולמה

**הצהרת IDFA:**
- אם משתמשים ב-Advertising Identifier
- אם יש פרסומות באפליקציה
- אם משתמשים ב-Facebook SDK

### שלב 7: שליחה לבדיקה

1. לחצו על "Submit for Review"
2. ענו על שאלות נוספות אם יש
3. המתינו לאישור

**זמני בדיקה ממוצעים:**
- בדיקה ראשונית: 24-48 שעות
- אם יש שאלות: +2-5 ימים
- סה"כ ממוצע: 1-7 ימים

## חלק 2: העלאה ל-Google Play

### שלב 1: פתיחת חשבון Google Play Console

1. היכנסו ל-[play.google.com/console](https://play.google.com/console)
2. התחברו עם חשבון Google
3. שלמו $25 (חד פעמי)
4. מלאו פרטי מפתח
5. החשבון פעיל מיידית

### שלב 2: יצירת אפליקציה חדשה

1. לחצו על "Create app"
2. מלאו:
   - App name
   - Default language
   - App or game
   - Free or paid
3. אשרו את ההצהרות

### שלב 3: Store Listing

**גרפיקה נדרשת:**

| אלמנט | גודל | הערות |
|-------|------|--------|
| App icon | 512 x 512 | PNG, ללא שקיפות |
| Feature graphic | 1024 x 500 | מופיע בראש העמוד |
| Phone screenshots | 320-3840 px | 2-8 צילומים |
| Tablet screenshots | 1080-7680 px | אם תומך בטאבלט |

**תיאורים:**

| שדה | אורך מקסימלי |
|-----|-------------|
| Short description | 80 תווים |
| Full description | 4,000 תווים |

### שלב 4: Content Rating

חובה למלא שאלון דירוג תוכן:
1. Production → Content Rating
2. Start questionnaire
3. ענו על כל השאלות בכנות
4. קבלו דירוג אוטומטי

**דירוגים אפשריים:**
- Everyone (כולם)
- Everyone 10+
- Teen
- Mature 17+
- Adults Only 18+

### שלב 5: Data Safety

**Google דורש לפרט:**
- אילו נתונים אתם אוספים
- למה אתם אוספים אותם
- האם משתפים עם צד שלישי
- האם אפשר למחוק נתונים

זה דף שנראה למשתמשים לפני ההורדה!

### שלב 6: בניית ה-APK/AAB

**Google מעדיף AAB (Android App Bundle):**

```bash
# ב-Android Studio
Build → Generate Signed Bundle / APK → Android App Bundle
```

או באמצעות Expo:
```bash
eas build --platform android
```

### שלב 7: העלאה לבדיקה

**מסלולי שחרור:**

| מסלול | מה זה | למי |
|-------|-------|-----|
| Internal testing | עד 100 בודקים | צוות פיתוח |
| Closed testing | קבוצה מוגדרת | בטא-טסטרים |
| Open testing | כל מי שרוצה | ציבור רחב |
| Production | כולם | משתמשים אמיתיים |

**תהליך מומלץ:**
1. Internal testing (1-2 ימים)
2. Closed testing (1-2 שבועות)
3. Production

**זמני בדיקה:**
- אפליקציה חדשה: 3-7 ימים
- עדכון: 1-3 ימים
- אחרי כמה אישורים: שעות בודדות

## סיבות נפוצות לדחייה (ואיך להימנע)

### Apple App Store

| סיבה | פתרון |
|------|--------|
| **Guideline 2.1 - App Completeness** | אל תשלחו עם באגים או תכונות שלא עובדות |
| **Guideline 2.3 - Accurate Metadata** | צילומי מסך חייבים להיות מהאפליקציה עצמה |
| **Guideline 4.2 - Minimum Functionality** | האפליקציה חייבת לעשות יותר מאתר |
| **Guideline 5.1 - Privacy** | חובה מדיניות פרטיות + הסכמה לאיסוף מידע |
| **Guideline 3.1 - Payments** | כל תשלום דיגיטלי חייב לעבור דרך In-App Purchase |

> ⚠️ **הכלל של Apple לתשלומים**: אם מוכרים תוכן דיגיטלי (מנוי, פיצ'רים, coins) - חייב In-App Purchase עם 30% עמלה ל-Apple. אם מוכרים מוצרים פיזיים (אוכל, מוצרים) - אפשר סליקה רגילה.

### Google Play

| סיבה | פתרון |
|------|--------|
| **Impersonation** | אל תשתמשו בשמות דומים לאפליקציות מפורסמות |
| **Misleading claims** | אל תבטיחו דברים שהאפליקציה לא עושה |
| **Data safety** | מלאו את כל הפרטים בצורה מדויקת |
| **Permissions** | בקשו רק הרשאות שבאמת צריך |
| **Target audience** | אם מיועד לילדים - עמדו בדרישות COPPA |

## טיפים מניסיון להעלאה מוצלחת

### 1. בדקו לפני שמעלים

- [ ] כל התכונות עובדות
- [ ] אין קריסות (crashes)
- [ ] האפליקציה מתאימה לגדלי מסך שונים
- [ ] הביצועים טובים (אין lag)
- [ ] כל הלינקים עובדים

### 2. צילומי מסך איכותיים

**טיפים:**
- השתמשו במכשיר אמיתי, לא סימולטור
- הסתירו את שורת הסטטוס או הראו 100% סוללה + WiFi מלא
- אל תראו נתונים אישיים
- הוסיפו טקסט שיווקי קצר מעל כל צילום

### 3. כתבו תיאור שמוכר

**מבנה מומלץ:**
```
שורה 1: מה האפליקציה עושה (הכי חשוב!)
שורה 2-3: למי מיועדת
שורות 4-10: תכונות עיקריות (עם אימוג'י ✓)
שורות אחרונות: CTA להורדה
```

### 4. ענו מהר לצוות הבדיקה

אם Apple או Google שולחים שאלות:
- ענו תוך 24 שעות
- היו ספציפיים ומנומסים
- צרפו צילומי מסך אם צריך
- אל תתווכחו - תקנו

### 5. התכוננו לדחייה

- זה קורה לכולם
- קראו את הסיבה בעיון
- תקנו את הבעיה (לא רק את הסימפטום)
- שלחו שוב עם הסבר מה תיקנתם

## App Store Optimization (ASO) - טיפים בסיסיים

**ASO = SEO לחנויות אפליקציות**

### מילות מפתח

**Apple (100 תווים):**
```
הזמנות,משלוחים,אוכל,מסעדות,מהיר,זול,דילים
```

**Google (נכלל בתיאור):**
- חזרו על מילות מפתח 3-5 פעמים בתיאור
- אל תעשו spam

### שם האפליקציה

- כללו מילת מפתח עיקרית אם אפשר
- דוגמה: "FoodExpress - הזמנת אוכל"
- לא יותר מדי ארוך

### דירוגים וביקורות

- בקשו מהמשתמשים לדרג (בזמן הנכון!)
- ענו לביקורות שליליות
- תקנו בעיות שמשתמשים מדווחים

## שאלות נפוצות

### כמה זמן לוקחת הבדיקה?

**Apple:** 24 שעות עד שבוע. אפליקציה ראשונה לוקחת יותר זמן.
**Google:** 3-7 ימים לאפליקציה חדשה, פחות לעדכונים.

### מה קורה אם האפליקציה נדחית?

מקבלים הסבר מפורט מה הבעיה. מתקנים ושולחים שוב. אין הגבלה על מספר הניסיונות.

### האם אפשר להעלות אפליקציה בחינם ואז לגבות?

כן, אבל:
- ב-Apple חייב להשתמש ב-In-App Purchase לתוכן דיגיטלי
- משתמשים שכבר הורידו בחינם צריכים להישאר עם גישה למה שהיה חינמי

### האם צריך אפליקציה נפרדת לישראל?

לא. אפליקציה אחת יכולה להיות זמינה בכל המדינות. אפשר להגביל לאזורים ספציפיים אם רוצים.

### מה ההבדל בין APK ל-AAB?

**APK** - קובץ התקנה מלא
**AAB** - פורמט חדש של Google שמייצר APK מותאם לכל מכשיר (יותר קטן, יותר מהיר)

Google דורש AAB לאפליקציות חדשות מ-2021.

## סיכום

העלאת אפליקציה לחנויות זה תהליך שדורש הכנה:

1. **הכינו את כל ה-assets מראש** - אייקון, צילומי מסך, תיאורים
2. **קראו את ה-Guidelines** - Apple ו-Google מפרסמים בדיוק מה הם רוצים
3. **בדקו לפני שמעלים** - אפליקציה יציבה = אישור מהיר
4. **היו מוכנים לתיקונים** - דחייה היא לא סוף העולם
5. **עקבו אחרי הביצועים** - ASO זה תהליך מתמשך
